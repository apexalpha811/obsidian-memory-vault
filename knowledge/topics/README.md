# topics/

Domain facts and reference knowledge about subjects the user cares about. `type: topic`.

**What to capture:** durable, non-obvious knowledge about a subject — the things worth not
re-deriving. Distinct from `decisions/` (choices) and `references/` (where to look): this is
*what is known* about a thing.

Lead with the subject and the key facts. Link related topics with `[[name]]`.
Filename: `<subject>.md`. Tag by domain.
