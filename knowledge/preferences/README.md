# preferences/

How the user likes things done — beyond the hard operating rules in `../../global/`.
`type: preference`.

**What to capture:** style and tone choices, preferred tools/workflows, formatting habits,
communication patterns. Things that, once known, mean the user shouldn't have to repeat them.

Lead with the rule, then **Why:** (reason given) and **How to apply:** (when it kicks in).
Filename: `<area>.md` (e.g. `writing-style.md`, `email-tone.md`). Tag by domain.
