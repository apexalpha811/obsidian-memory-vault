# tools-and-setup/

Environment and tooling memory — what's installed, how it's wired, and the gotchas.
`type: tool`.

**What to capture:** install location, how it's configured, non-obvious gotchas and their
fixes, verification commands. This is the home for the kind of setup notes currently living
only as `../../meta/changelog.md` bullets (liteparse, Tesseract OCR, obsidian-mcp, etc.) —
migrate them here so they're findable, and keep the changelog as the dated record of *when*.

Lead with what the tool is and its state, then **Setup:**, **Gotchas:**, **Verify:**.
Filename: `<tool>.md`. Tag by domain (most are `[work]`).
